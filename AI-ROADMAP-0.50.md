# 離線字幕工廠 AI Roadmap：0.40 → 0.50

> 建立日期：2026-07-17  
> 起始版本：0.40.0  
> 目標版本：0.50.0  
> 目標定位：把 0.40 的雲端 AI 字幕優化 MVP 推進為可恢復、可稽核、可撤銷，並正式支援本機 LLM 的穩定版本。

## 一、執行原則

1. AI 功能維持預設停用，原有 Whisper、修剪、校閱與輸出必須可完全離線運作。
2. AI 不得直接覆蓋字幕；所有修改先形成建議，經確認後才寫入校閱字幕。
3. cue ID、順序與時間碼預設由程式鎖定，AI 只負責文字內容。
4. 每個版本必須有不消耗外部 API 的 mock 測試，以及既有 `npm run check` 回歸測試。
5. API Key、Prompt 內容及外部服務原始回應不得進入一般工作日誌或錯誤紀錄。
6. 每個里程碑完成後更新 `NEXT-VERSION-FIX-LOG.md`，通過驗收才進入下一版。

## 二、版本里程碑

### 0.41.0：請求可靠性與錯誤恢復

目標：長字幕遇到網路不穩、限流或單批失敗時，不必整份重做。

- 實作 HTTP 408、429、500、502、503、504 自動重試。
- 採指數退避與隨機抖動，尊重 `Retry-After`。
- 加入供應商並行數限制，預設同一任務一次只處理一批。
- 保存批次 checkpoint；失敗後可從未完成批次繼續。
- 區分可重試、不可重試、使用者取消與驗證失敗。
- UI 顯示目前批次、重試次數、等待秒數與錯誤摘要。
- APP 關閉或 server 重啟後，能辨識未完成任務並提供恢復或放棄。

完成條件：

- mock provider 可模擬 429、逾時、斷線及成功恢復。
- 已完成批次不重送，取消後不再產生後續請求。
- 原字幕在任務完成並確認前保持不變。

### 0.42.0：術語表、Prompt 範本與多段選取（已完成）

目標：讓校閱人員能針對課程、教師與領域建立可重用規範。

- 建立專案術語表 JSON 格式，支援原詞、標準詞、大小寫、禁止翻譯及備註。
- 支援 CSV／JSON 匯入與匯出。
- 建立 Prompt 範本管理，分為錯字標點、斷句、術語、贅詞與翻譯。
- 自訂 Prompt 與內建安全約束分離，避免覆蓋 cue 保護規則。
- 字幕列表加入多選、連續範圍選取及「只處理搜尋結果」。
- 顯示預計傳送 cue 數量及大約批次數。

完成條件：

- 術語表在批次切分後仍一致套用。
- 多選處理只回傳選取 cue，其他字幕完全不變。
- Prompt 範本可保存至專案並隨專案檔重新載入。

### 0.43.0：優化報告與一鍵撤銷（已完成）

目標：所有 AI 修改都可以追溯、比較及完整復原。

- 每次優化建立 session ID。
- 保存來源字幕快照、建議、接受／略過狀態、模型、模式、批次與時間。
- 產出 `suggestions.json`、`optimized-preview.srt` 與 Markdown 摘要報告。
- 新增「撤銷本次 AI 優化」與「重新套用」功能。
- 撤銷只影響該 session 接受的文字，不回退之後的人工修改；衝突時先提示。
- 報告遮蔽 API Key、Authorization header 與敏感錯誤內容。

完成條件：

- 全部接受後可一鍵還原。
- APP 重啟後仍可查看報告並撤銷未衝突修改。
- 報告能正確統計處理、修改、接受、略過與失敗數量。

### 0.44.0：作業系統安全金鑰與隱私控制（已完成）

目標：把 0.40 的獨立 secrets 檔案提升為正式桌面憑證保存。

- macOS 使用 Keychain／Electron `safeStorage`。
- Windows 使用 DPAPI／Credential Manager 或 Electron `safeStorage`。
- 舊 `ai-secrets.json` 提供一次性遷移，成功後安全移除舊金鑰。
- 增加清除金鑰、替換金鑰與金鑰來源狀態。
- 第一次啟用雲端 AI 時顯示資料傳送同意。
- 清楚列出將傳送的字幕範圍，不傳影片與音訊。

完成條件：

- 一般設定、專案檔、日誌與匯出包均找不到 API Key。
- macOS 與 Windows 安裝版完成保存、讀取、替換、清除與遷移測試。

### 0.45.0：多供應商 Adapter（已完成）

目標：建立真正可替換的 AI provider 層。

- 統一 provider interface：連線測試、模型列表、文字優化、取消、錯誤分類。
- 正式支援 OpenAI、OpenAI-compatible、Azure OpenAI。
- 評估並加入 Anthropic Claude、Google Gemini，若 schema 不穩定可延至 0.51。
- 各供應商設定彼此隔離，切換時不覆蓋其他金鑰。
- capabilities 宣告是否支援 JSON schema、串流、模型列表及本機端點。

完成條件：

- 共用 contract tests 可套用所有 provider。
- 任一 provider 回傳格式變化不影響字幕驗證與校閱 UI。

### 0.46.0：翻譯與雙語字幕

目標：從單語文字替換升級為保留原文的雙語字幕流程。

- cue 資料增加 `sourceText`、`translatedText`，不再只保存單一 text。
- 支援原文在上／譯文在下及譯文在上／原文在下。
- 支援雙語 SRT、VTT 與 ASS 輸出。
- 可個別編輯原文和譯文，並設定雙語字型、大小與間距。
- 翻譯不得改變 cue 時間碼與數量。

完成條件：

- 單語舊專案能無損載入。
- 雙語字幕預覽與 FFmpeg／ASS 實際輸出一致。

### 0.47.0：低可信片段工作流

目標：只把需要處理的片段交給 AI，降低費用及人工檢查量。

- Whisper.cpp 轉錄結果保存可取得的 confidence、no-speech probability 或替代品質指標。
- 建立低可信、過長、速度過快、重複文字及疑似專有名詞等規則。
- 校閱頁提供問題篩選與批次選取。
- AI 預設只接收字幕文字與前後文；音訊不上傳。
- 若引擎無 confidence，使用規則型風險分數並明確標示來源。

完成條件：

- 低可信片段可重現篩選，且不會自動修改高可信字幕。
- 品質指標缺失時不偽造 confidence。

### 0.48.0：本機 LLM 基礎支援

目標：提供不將字幕傳出電腦的 AI 優化方案。

- 支援 Ollama 與 LM Studio 的 OpenAI-compatible 本機端點。
- 自動偵測常見 localhost 端點並列出模型。
- 提供本機／雲端狀態標示與隱私差異。
- 模型能力檢查：context 長度、JSON 輸出及繁體中文品質。
- 本機模型使用較小批次與更嚴格回應修復策略。
- 不自動下載大型模型；由使用者明確選擇安裝方式與儲存位置。

完成條件：

- Ollama 與 LM Studio 各完成至少一個模型的端到端測試。
- 斷網狀態可完成本機字幕優化。

### 0.49.0：跨平台 Beta 與效能硬化

目標：凍結功能，集中處理效能、相容性與發布品質。

- 進行 Windows 10／11 x64、macOS Apple Silicon 實機驗收。
- 建立 10、60、120 分鐘字幕壓力測試。
- 驗證大量 cue 的記憶體、UI 捲動、checkpoint 與撤銷效能。
- 完成設定遷移、舊專案相容與損壞資料復原測試。
- 清理 provider 錯誤訊息及中文操作引導。
- 完成 Windows 簽章、macOS Developer ID／notarization 發布檢查表。

完成條件：

- 無阻斷級資料遺失、安全或時間碼錯誤。
- 全套單元、核心 API、封裝與實機測試通過。

### 0.50.0：本機 AI 穩定版

目標：正式發布兼具雲端與本機 LLM 的可稽核 AI 字幕校閱版本。

- 完成 0.41～0.49 功能整合與文件。
- 首次啟動提供「完全離線」、「本機 LLM」、「雲端 AI」三種模式說明。
- 完成資料傳送、金鑰、模型、報告及清除資料說明。
- 發布 Windows Setup／Portable 與 macOS DMG／ZIP。
- 產出 SHA-256、第三方授權、Release Notes 與升級說明。
- GitHub tag、Release、Actions artifact 與來源分支一致。

發布門檻：

- `npm run check`、provider contract tests、封裝驗證及跨平台實機驗收全部通過。
- 本機 LLM 斷網端到端流程通過。
- 雲端 AI 未啟用時不得產生任何外部 AI 請求。
- cue ID、數量與時間碼保護測試零失敗。

## 三、建議時程

以每個里程碑 3～5 個有效工作日估算：

| 階段 | 版本 | 預估工作量 | 累計 |
|---|---|---:|---:|
| 穩定性 | 0.41 | 4～5 日 | 第 1 週 |
| 校閱工具 | 0.42～0.43 | 7～9 日 | 第 2～3 週 |
| 安全與供應商 | 0.44～0.45 | 7～10 日 | 第 4～5 週 |
| 雙語與品質選取 | 0.46～0.47 | 7～10 日 | 第 6～7 週 |
| 本機 LLM | 0.48 | 5～7 日 | 第 8 週 |
| Beta 與發布 | 0.49～0.50 | 6～9 日 | 第 9～10 週 |

總估算：約 36～50 個有效工作日；實際時程取決於跨平台簽章、公證、供應商測試帳號及本機模型硬體。

## 四、優先風險

1. Provider 回傳格式差異：以 adapter contract tests 與後端 schema 驗證隔離。
2. 長任務中斷造成重複費用：0.41 優先完成 checkpoint、retry 與冪等批次 ID。
3. 人工修改與撤銷衝突：0.43 以 session 快照與文字版本比對處理。
4. API Key 洩漏：0.44 完成 OS 安全儲存，發布前掃描設定、日誌與封裝內容。
5. 本機模型輸出不穩：限制支援模型、縮小批次並保留嚴格 cue 驗證。
6. 雙語資料模型破壞舊專案：先做資料遷移與向下相容測試，再開放輸出。

## 五、近期下一步

0.41～0.45 已完成。下一個里程碑為 0.46.0 翻譯與雙語字幕資料模型；開始前需先建立單語舊專案的無損遷移測試。
