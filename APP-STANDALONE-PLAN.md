# 離線字幕工廠：獨立單機版 APP 規劃與 UI 整併建議

> 建立日期：2026-07-15
> 規劃基礎：Electron + 本機 Node server + FFmpeg + Python/Whisper + HTML/CSS/JavaScript renderer
> 產品目標：使用者安裝後即可完成影片轉錄、字幕校對與影片輸出，不需要自行安裝 Python、FFmpeg、Whisper、模型或執行批次檔。

---

## 一、產品定位與交付原則

獨立單機版應被視為「完整產品」，不是開發環境的啟動器。安裝、更新、工具修復與模型管理都應由 APP 負責。

核心原則：

1. 使用者只需要安裝一個正式安裝程式。
2. 基礎轉錄能力在離線狀態下也能使用。
3. 不要求使用者開啟命令列、執行 `.bat`、使用 winget 或自行設定 PATH。
4. 工具檢查失敗時，APP 顯示「自動修復」，而不是提供人工安裝說明。
5. 大型模型可在 APP 內按需下載，但下載、校驗、續傳與移除都由 APP 管理。
6. 所有影片、字幕、模型與暫存資料都清楚顯示儲存位置與容量。

## 二、建議的單機版架構

### 2.1 應用程式層

- Electron main process
  - 視窗、選檔、資料夾、更新、安裝與權限管理。
  - 啟動及關閉本機服務。
  - 管理 runtime、模型與工具修復。
- Renderer
  - 首頁／專案列表。
  - 新增任務精靈。
  - 處理佇列與進度。
  - 字幕校對工作區。
  - 輸出設定與結果。
  - 系統診斷與儲存空間管理。
- Local API server
  - 任務、上傳、轉錄、校對、輸出與狀態 API。
  - 僅監聽 `127.0.0.1`，使用隨機 token 驗證。

### 2.2 隨 APP 交付的必要 runtime

正式 Windows 安裝包至少應包含：

- 可攜式 Node runtime。
- FFmpeg 與 FFprobe。
- 可攜式 Python runtime。
- 已安裝且經驗證的 Whisper runtime。
- CPU 可用的基礎模型，例如 `small` 或產品測試後選定的預設模型。
- 字型 fallback 與授權允許的必要資源。
- `tools/manifest.json`：版本、hash、大小、來源與相容性資訊。

使用者電腦上的全域 Python、FFmpeg 或 PATH 只能作為開發／進階選項，不應成為正式版必要條件。

### 2.3 建議的資料目錄

```text
%LOCALAPPDATA%/OfflineSubtitleFactory/
├── app-runtime/        # APP 管理的 runtime
├── models/             # Whisper 模型
├── jobs/               # 專案與任務
├── cache/              # 可清除暫存
├── logs/               # 診斷日誌
├── backups/            # 校對版本快照
└── config/             # 使用者設定
```

安裝目錄盡量保持唯讀；會變動的模型、任務與設定放在 userData，避免權限與升級覆蓋問題。

## 三、首次啟動與自動安裝／修復流程

### 3.1 啟動狀態機

```text
啟動 APP
  → 驗證 manifest
  → 驗證 Node / FFmpeg / FFprobe / Python / Whisper
  → 驗證預設模型
  → 驗證可寫入空間與版本相容性
  → 全部正常：進入首頁
  → 可修復：顯示 APP 內修復進度
  → 無網路：使用 bundled fallback 或顯示離線修復媒體選項
  → 無法修復：產生診斷包，提供重新安裝按鈕
```

### 3.2 工具來源策略

依優先順序：

1. APP 自帶且 hash 正確的 runtime。
2. APP 管理目錄中已下載的新版本 runtime。
3. 僅在進階設定啟用時，才允許使用系統既有工具。

不建議自動修改使用者 PATH，也不建議在背景替使用者安裝全域 Python。

### 3.3 自動修復內容

APP 偵測到缺少或損壞時應自動執行：

- 建立必要目錄。
- 從安裝包 resources 解壓缺少的 runtime。
- 驗證 SHA-256。
- 執行最小 smoke test：
  - `ffmpeg -version`
  - `ffprobe -version`
  - `python --version`
  - `python -c "import whisper"`
  - 使用短音訊進行一次轉錄 smoke test。
- 寫入安裝結果與版本資訊。
- 失敗時回滾到上一個可用版本。

介面只顯示：

- 「正在準備離線轉錄元件」
- 個別元件進度
- 下載大小／剩餘時間
- 「暫停」「重試」「查看詳細資料」

不再顯示「請安裝 Python」「請執行 winget」或「開啟安裝腳本」作為主要流程。

### 3.4 模型管理

建議將 runtime 與模型分開：

- 安裝包內附一個能立即使用的預設模型。
- medium／large-v3 等大型模型在「模型管理」中按需下載。
- 顯示語言適用性、品質、速度、容量與建議硬體。
- 支援下載續傳、hash 驗證、暫停、刪除與移動模型目錄。
- 磁碟不足時先提示需要容量，不要下載到一半才失敗。

### 3.5 GPU 與硬體相容性

- 預設必須保證 CPU 能執行。
- 偵測 NVIDIA CUDA 能力後，再選擇相容 runtime，不直接假設所有 NVIDIA GPU 都可用。
- GPU runtime 不應阻塞首次使用，可於背景提供「安裝 GPU 加速」選項。
- 顯示預估速度差異與下載容量，讓使用者決定是否安裝。
- GPU 初始化失敗時自動回退 CPU，不讓整個任務失敗。

## 四、APP 功能與頁面規劃

### 4.1 首頁／專案中心

首頁只承擔三件事：

- 新增字幕專案。
- 繼續最近專案。
- 查看處理中任務。

建議內容：

- 「新增專案」主按鈕。
- 最近專案卡片：影片縮圖、名稱、時長、狀態、最後編輯時間。
- 處理佇列：排隊、轉錄、輸出、失敗與剩餘時間。
- 儲存空間簡要提示。
- 工具異常時顯示一條可展開的健康狀態列，不以全畫面遮罩阻擋正常可用功能。

### 4.2 新增專案精靈

將目前一頁內的大量設定拆為 3 步：

1. 選擇來源
   - 影片／音訊。
   - 可選既有 SRT。
   - 顯示檔案、時長、解析度、音軌與容量。
2. 選擇轉錄方式
   - 語言。
   - 快速／平衡／精準。
   - 模型建議由 APP 自動決定，高級設定收合。
3. 字幕規則與輸出
   - 套用規則範本。
   - 每行字數、標點、繁簡與術語表。
   - 建立後先轉錄，輸出格式留到輸出階段設定。

### 4.3 處理中心

- 全部任務與佇列排序。
- 目前階段、真實進度、處理速度與剩餘時間。
- 暫停／取消／重新執行。
- 失敗時顯示可理解的原因與「自動修復後重試」。
- 詳細 log 預設收合，供診斷使用。

### 4.4 字幕校對工作區

建議使用三區結構：

```text
┌──────────────── 頂部專案列 ────────────────┐
│ 返回 / 專案名稱 / 儲存狀態 / 匯出          │
├───────────────┬────────────────────────────┤
│ 影片＋波形時間軸 │ 字幕清單與文字編輯          │
│ 播放控制        │ 搜尋 / 篩選 / 待檢查          │
├───────────────┴────────────────────────────┤
│ 目前字幕時間、快捷鍵提示或屬性面板           │
└────────────────────────────────────────────┘
```

功能整併：

- 「載入影片」「載入 SRT」「載入專案預設」移到專案選單，不占主要頂部空間。
- 「下載 SRT」「儲存校稿包」「燒錄輸出」整併為一個「匯出」主按鈕。
- 「字幕校稿／燒錄預覽」改為工作區分頁，避免與播放控制混在一起。
- 全片批次時間調整放入「批次工具」選單。
- 預設顯示自動保存狀態，不需要持續提供「另存 SRT」主按鈕。
- 增加波形、上一句／下一句、待檢查導航、復原／重做與快捷鍵說明。

### 4.5 輸出中心

- 字幕檔：SRT、VTT、ASS。
- 軟字幕影片。
- 硬燒錄影片。
- 解析度、編碼、品質、字幕樣式與輸出目錄。
- 顯示預估輸出大小與時間。
- 完成後提供「開啟檔案」「開啟資料夾」「再次輸出」。

### 4.6 設定與維護

設定分成四區：

- 一般：語言、預設專案位置、暫存位置。
- 轉錄：預設模式、模型、CPU threads、GPU 加速。
- 儲存：模型容量、任務容量、清除快取、備份保留天數。
- 診斷：runtime 版本、健康檢查、自動修復、匯出診斷包。

## 五、目前 UI 問題與整併方向

### 5.1 首次啟動遮罩

目前問題：

- 顯示 FFmpeg、Python、Whisper 未安裝，並要求使用者執行腳本與 winget。
- 「打開安裝腳本」仍把技術流程交給非技術使用者。
- 全畫面遮罩讓使用者無法理解 APP 主體，也無法先匯入既有 SRT。

建議：

- 改為「正在準備離線元件」自動流程。
- 只在必須下載時詢問下載容量與位置。
- 允許不需 ASR 的既有 SRT 校對功能先進入。
- 技術細節放在「詳細資料」，主畫面只顯示可處理的行動。

### 5.2 主畫面資訊過量

目前左側列出 10 個步驟，但實際使用者不是逐頁完成十次操作；很多只是同一任務的後端階段。這會讓流程看起來比實際更複雜。

建議整併成四個產品階段：

1. 匯入。
2. 轉錄。
3. 校對。
4. 輸出。

「環境檢查」「套用規則」應是系統狀態或處理子階段，不應占據主要導航。

### 5.3 表單設定層級不清楚

目前語言、ASR 引擎、模型字串、效能模式、CPU threads、輸出格式同時呈現。一般使用者難以判斷哪些需要修改。

建議：

- 一般模式只顯示語言與快速／平衡／精準。
- 模型、CPU threads、ASR 引擎移到「進階設定」。
- 不顯示尚未真正支援的引擎選項，避免選擇 faster-whisper／whisper.cpp 後實際仍走 Python Whisper。
- 輸出格式移到輸出中心，不放在建立任務階段。

### 5.4 虛假的效能資訊

目前畫面預設顯示 CPU 18%、記憶體 42% 等固定 meter，容易被理解為即時數據。

建議：

- 有真實監測資料才顯示數值。
- 沒有資料時顯示「尚未開始」而不是模擬百分比。
- 一般使用者只需看到處理速度與剩餘時間；CPU/GPU 細節移到展開區。

### 5.5 校對頁頂部操作過密

目前頂部同時出現載入、儲存、燒錄、返回；影片下方又同時出現播放、階段切換、下載與清理規則。主動作層級不明確。

建議：

- 頂部只留：返回、專案名稱、保存狀態、匯出。
- 播放控制集中在影片／時間軸下方。
- 校對工具集中在右側字幕清單上方。
- 樣式與燒錄設定做成第二分頁或右側屬性抽屜。
- 危險或批次操作放入更多選單，避免誤觸。

### 5.6 校對頁空間分配

目前影片與字幕清單約各半，但左側影片下方還有大塊「目前字幕」面板，右側字幕工具列也會換行，占用可用高度。

建議：

- 目前字幕直接疊加於影片預覽，或縮成時間軸下方單行。
- 右側工具列拆成搜尋列與批次工具選單。
- 支援拖曳調整左右欄寬。
- 小螢幕改為影片／字幕上下分頁，不強制雙欄。

## 六、建議開發優先順序

### P0：真正可交付的單機版

1. 建立正式 portable runtime 與 manifest／hash。
2. 安裝包內附 CPU 可用的 FFmpeg、Python、Whisper 與預設模型。
3. Electron 內建自動檢查、解壓、修復與 smoke test。
4. 移除使用者手動執行 `.bat`／winget 的主要流程。
5. 完成乾淨 Windows VM 的首次安裝、離線啟動、轉錄、校對與輸出測試。

### P1：產品流程整併

1. 首頁改為專案中心。
2. 建立三步新增專案精靈。
3. 左側十步流程縮為匯入／轉錄／校對／輸出。
4. 校對頁重新整理主動作與版面。
5. 建立統一輸出中心。

### P2：維護與模型管理

1. APP 內模型下載、續傳、驗證與移除。
2. GPU 加速元件按需安裝與 CPU fallback。
3. 儲存空間管理與自動清理。
4. 自動更新與 runtime 回滾。
5. 一鍵匯出診斷包。

## 七、正式發版驗收條件

- 全新 Windows 使用者不安裝 Python、FFmpeg、Node 或 Whisper，也能完成第一次字幕轉錄。
- 斷網狀態下，安裝包內附的預設模型可正常完成轉錄。
- 元件損壞時可以在 APP 內自動修復，不要求使用命令列。
- GPU 不相容時自動回退 CPU。
- 安裝、免安裝、升級與解除安裝不會刪除使用者專案。
- 首頁不再被技術性安裝說明永久阻擋。
- 一般使用者不需要理解 ASR 引擎、PATH、Python venv 或 FFmpeg 參數。
- 主要流程在四個產品階段內完成：匯入、轉錄、校對、輸出。
- 校對內容自動保存，APP 異常關閉後可恢復。
- 以短片、兩小時長片、中文字幕、英文字幕、CPU-only、CUDA、低磁碟空間與無網路環境完成驗收。
