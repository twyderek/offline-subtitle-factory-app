# 離線字幕工廠：簡易影片修剪功能設計與製作規劃

> 規劃日期：2026-07-16
> 規劃基礎：Electron + 本機 Node API + FFmpeg／FFprobe + HTML5 Video + 真實音訊波形
> 本文件目前為設計規格，不代表功能已實作。

---

## 一、功能定位

本功能定位為「字幕製作前後可使用的簡易修剪」，不是取代 Final Cut Pro、Premiere 或 DaVinci Resolve 的完整剪輯器。

第一版只解決最常見的需求：

- 剪掉影片開頭不需要的等待或測試畫面。
- 剪掉影片結尾多餘內容。
- 保留影片中一段連續區間。
- 修剪後直接送入離線轉錄、字幕校對與輸出流程。
- 已有字幕時，能安全地重新計算字幕時間碼。

第一版暫不納入：

- 多軌影片／音訊編輯。
- 轉場、濾鏡、字幕特效、貼圖或背景音樂。
- 任意多段素材拼接。
- 複雜調色與音訊混音。

這樣能控制開發風險、維持介面簡單，也符合目前「離線字幕工具」的產品定位。

## 二、建議的功能範圍

### 2.1 第一階段：單一區間修剪（MVP）

使用者在時間軸設定：

- 起點 In。
- 終點 Out。
- 保留 In 到 Out 之間的影片。

必要功能：

- 播放、暫停、前後 5 秒。
- 以目前播放位置設定起點／終點。
- 拖曳左右修剪把手。
- 直接輸入精確時間。
- 循環播放選取區間。
- 顯示原始長度、保留長度與刪除長度。
- 重設為完整影片。
- 套用修剪與取消處理。
- 修剪後可還原原始影片。

### 2.2 第二階段：刪除中間片段

在 MVP 穩定後才加入：

- 建立多個保留區間。
- 刪除影片中間的停頓或錯誤段落。
- 自動重新排列所有後續字幕時間碼。
- 顯示剪接點與片段清單。

### 2.3 第三階段：簡易組合

若使用者確實有需求，再評估：

- 合併多個影片。
- 淡入淡出。
- 音量正規化。
- 靜音指定片段。

## 三、介面設計

### 3.1 功能入口

建議提供三個入口，但共用同一個修剪工作區：

1. 新增專案、選擇影片後顯示「先修剪影片」按鈕。
2. 專案卡片的更多選單加入「修剪影片」。
3. 字幕校對左側導覽在「載入影片」與「校對」之間加入「修剪」。

不建議在首頁新增一個獨立的大型功能卡，以免首頁再次變得複雜。

### 3.2 修剪工作區版型

```text
┌─────────────────────────────────────────────────────┐
│ ← 返回專案   修剪影片   原長 12:46   已自動儲存       │
├──────────────────────────────┬──────────────────────┤
│                              │ 修剪設定              │
│          影片預覽             │ 起點  00:00:12.500    │
│                              │ 終點  00:11:38.200    │
│                              │ 保留  00:11:25.700    │
├──────────────────────────────┤                      │
│ 真實音訊波形                  │ [設為起點] [設為終點] │
│ [灰色刪除區][藍色保留區][灰色] │ [循環預覽] [重設]     │
│          ▲播放游標            │                      │
├──────────────────────────────┤ [取消] [套用修剪]      │
│  −5秒    ▶播放    5秒    目前時間 │                      │
└──────────────────────────────┴──────────────────────┘
```

### 3.3 視覺規則

- 沿用目前深藍側欄、白色工作區與亮藍主操作按鈕。
- 保留區間使用藍色半透明遮罩。
- 會刪除的區域使用深灰半透明遮罩，不真的移除原始檔。
- 起點使用綠色把手，終點使用橘色把手，並在上方顯示時間。
- 套用前顯示：「原始影片不會被覆蓋」。
- 正在輸出修剪影片時顯示 FFmpeg 真實進度、剩餘時間與取消按鈕。

### 3.4 操作方式

- `I`：設定起點。
- `O`：設定終點。
- `Space`：播放／暫停。
- `←`／`→`：小幅移動播放位置。
- `Shift + ←`／`Shift + →`：移動修剪點。
- 拖曳把手時，影片預覽跟隨至對應畫面。
- 起點不得大於或等於終點。
- 最短保留長度建議為 0.5 秒。

## 四、與目前架構的整合方式

### 4.1 非破壞式處理

不得覆蓋：

```text
jobs/<jobId>/input/original.mp4
```

新增：

```text
jobs/<jobId>/
├── input/
│   └── original.mp4
├── working/
│   ├── edit-plan.json
│   ├── media-trimmed.partial.mp4
│   ├── media-trimmed.mp4
│   ├── waveform-640.json
│   └── home-thumbnail.jpg
└── review-output/
    └── trimmed.srt
```

`edit-plan.json` 建議格式：

```json
{
  "version": 1,
  "mode": "single-range",
  "sourceDuration": 766.4,
  "in": 12.5,
  "out": 698.2,
  "outputDuration": 685.7,
  "strategy": "precise",
  "createdAt": "2026-07-16T00:00:00.000Z"
}
```

原始影片永遠保留；還原時只要停用 edit plan，不需要重新上傳影片。

### 4.2 有效影片路徑

目前後端多數功能直接呼叫 `getVideoPath(job)` 取得 input 原始影片。

實作修剪後應新增：

```text
getOriginalVideoPath(job)
getEffectiveVideoPath(job)
```

`getEffectiveVideoPath(job)` 的解析順序：

1. 已完成且驗證成功的 `working/media-trimmed.mp4`。
2. 原始 `input` 影片。

以下功能都必須改用有效影片：

- 音訊前處理與 Whisper 轉錄。
- 影片資訊與時長。
- 真實聲波。
- 首頁縮圖。
- 校閱頁影片。
- 軟字幕與硬字幕輸出。

### 4.3 修剪前後的工作流程

建議主要流程：

```text
選擇影片
  → 瀏覽器直接預覽，不產生新檔
  → 使用者設定 In／Out
  → 建立字幕任務並保存 edit plan
  → FFmpeg 產生 trimmed media
  → 對 trimmed media 執行 Whisper
  → 字幕校對
  → 匯出
```

這比「完整轉錄後才修剪」更省時間，也不需要再次執行 Whisper。

既有專案則使用：

```text
載入既有專案
  → 設定 In／Out
  → 產生 trimmed media
  → 重新計算現有字幕時間
  → 使用者確認邊界字幕
  → 儲存新的校對版本
```

## 五、FFmpeg 處理策略

### 5.1 快速修剪

特性：

- 使用 stream copy。
- 幾乎不重新編碼，速度快且不會額外損失畫質。
- 起點可能受到關鍵影格限制，不保證逐幀精準。

概念指令：

```text
ffmpeg -ss <in> -i input.mp4 -t <duration> -map 0 -c copy \
  -avoid_negative_ts make_zero -movflags +faststart output.mp4
```

適合：只需要大致剪掉片頭、片尾。

### 5.2 精準修剪

特性：

- 重新編碼影片，可精確到指定時間。
- 耗時較長，會產生新的影片編碼。
- 音訊建議輸出 AAC，避免不同來源格式的時間戳問題。

macOS Apple Silicon 優先測試 `h264_videotoolbox`，失敗時回退 `libx264`；Windows 先以 `libx264` 保證相容性，再評估硬體加速。

概念指令：

```text
ffmpeg -ss <in> -i input.mp4 -t <duration> \
  -map 0:v:0 -map 0:a:0? \
  -c:v h264_videotoolbox -c:a aac -b:a 192k \
  -avoid_negative_ts make_zero -movflags +faststart output.partial.mp4
```

完成並通過 FFprobe 驗證後，再以原子重新命名方式改為 `media-trimmed.mp4`。

### 5.3 建議預設

- UI 預設「精準修剪」。
- 進階選項提供「快速修剪」。
- 顯示簡單說明：精準模式邊界準確；快速模式速度快，但可能多保留鄰近畫面。

## 六、字幕時間軸處理

### 6.1 修剪發生在轉錄前

Whisper 直接處理修剪後影片，因此字幕從 00:00:00 開始，不需額外轉換。

### 6.2 修剪發生在已有字幕之後

單一保留區間 `[in, out]` 的字幕轉換規則：

```text
newStart = max(originalStart, in) - in
newEnd   = min(originalEnd, out) - in
```

處理原則：

- 完全位於保留區間外的字幕刪除。
- 跨越起點或終點的字幕截短。
- 截短後小於 0.1 秒的字幕刪除。
- 重新編號所有字幕。
- 邊界受到截短的字幕標記為「待檢查」。
- 原字幕另存備份，不覆蓋。

第二階段若支援多區間，需建立累積時間偏移表，不能只做固定減法。

## 七、後端 API 規劃

建議新增：

```text
GET    /api/jobs/:jobId/edit-plan
PUT    /api/jobs/:jobId/edit-plan
POST   /api/jobs/:jobId/apply-trim
GET    /api/jobs/:jobId/trim-status
POST   /api/jobs/:jobId/cancel-trim
DELETE /api/jobs/:jobId/trim
```

職責：

- `edit-plan`：讀取／驗證／儲存非破壞式修剪設定。
- `apply-trim`：啟動 FFmpeg，產生有效影片。
- `trim-status`：回傳進度、階段、剩餘時間與錯誤。
- `cancel-trim`：中止 FFmpeg 並刪除 `.partial`。
- `DELETE trim`：還原使用原始影片，不刪除原始輸入。

後端新增 `runningTrims` 與 `AbortController`；不可和轉錄或燒錄同時修改同一任務。

## 八、狀態、快取與一致性

新增階段名稱：

- `trim-planned`
- `trim-preparing`
- `trimming`
- `trim-validating`
- `trim-completed`
- `trim-cancelled`
- `trim-failed`

套用或還原修剪後必須失效：

- 舊聲波 JSON。
- 舊首頁縮圖。
- 舊 probe 資訊。
- 尚未輸出的暫存字幕。

如果已有字幕，必須先完成字幕時間重算，才能把狀態標成可校閱。

## 九、效能與安全需求

- 預覽階段只移動 HTML5 Video 時間，不啟動 FFmpeg。
- 使用者按下「套用修剪」後才產生新影片。
- 修剪前檢查輸出目錄可寫入與剩餘空間。
- 至少保留預估輸出大小加 1 GB 安全空間。
- 使用 `.partial` 檔，成功後才更名，避免中斷留下假成品。
- FFmpeg 參數以陣列傳入 `spawn`，不可組合 shell 字串。
- 所有 In／Out 必須在伺服器再次驗證，不信任前端數值。
- App 關閉或任務取消後清除未完成的 partial 檔案。
- 同一任務同時間只能執行修剪、轉錄或燒錄其中一項。

## 十、測試規劃

### 10.1 單元測試

- In／Out 範圍驗證。
- 時間格式轉換。
- 單一區間字幕重算。
- 跨越邊界字幕截短。
- 完全位於區間外字幕刪除。
- 取消與還原 edit plan。

### 10.2 FFmpeg 整合測試

建立 10 秒測試影片，保留 2–7 秒：

- 輸出長度約為 5 秒。
- 影片與音訊軌均存在。
- 第一個時間戳接近 0。
- FFprobe 可正常讀取。
- 快速與精準模式都能完成。

### 10.3 字幕整合測試

- 0–10 秒影片包含 5 段字幕。
- 修剪 2–7 秒後，只保留重疊字幕。
- 新字幕從 0 秒開始。
- 邊界字幕被標記待檢查。
- 修剪後校閱自動跟隨仍正確。

### 10.4 封裝版測試

- macOS Apple Silicon DMG 安裝版。
- Windows x64 安裝版與可攜版。
- 無系統 FFmpeg／Python 的乾淨環境。
- 取消修剪、空間不足、唯讀目錄與損壞影片。
- 修剪後轉錄、校閱、軟字幕與硬字幕輸出完整流程。

## 十一、建議製作順序

### P0：規格與資料層

1. 定義 `edit-plan.json`。
2. 將影片來源抽象為 `getEffectiveVideoPath(job)`。
3. 建立字幕單區間重算函式與單元測試。

### P1：MVP 介面與預覽

4. 建立修剪工作區。
5. 重用現有影片播放器與真實聲波。
6. 完成 In／Out 把手、時間輸入、循環預覽與快捷鍵。

### P2：實際修剪

7. 建立 apply／status／cancel／restore API。
8. 完成精準 FFmpeg 修剪與進度。
9. 完成 partial、FFprobe 驗證、原子更名與快取失效。

### P3：字幕與流程整合

10. 新專案在轉錄前套用修剪。
11. 既有字幕執行時間重算與邊界警告。
12. 校閱、縮圖、聲波與輸出全部改用有效影片。

### P4：驗收

13. 自動測試、真實長影片測試與效能量測。
14. macOS／Windows 封裝後 smoke test。
15. 更新功能說明、快捷鍵與修正日誌。

## 十二、工作量與風險評估

### 第一版工作量

- UI／互動：中。
- FFmpeg 處理：中。
- 字幕時間軸：中高。
- 封裝與跨平台測試：中。

### 主要風險

1. Stream copy 起點不精準：以精準模式作預設並清楚標示快速模式限制。
2. 修剪後字幕不同步：所有下游功能必須統一使用 effective media，並以自動測試覆蓋時間重算。
3. 重複編碼降低畫質：保留原始檔、提供快速模式，最終硬字幕輸出避免不必要的多次編碼。
4. 長影片佔用大量空間：套用前預估空間，支援取消與還原。
5. 轉錄、修剪、燒錄互相衝突：加入任務鎖與明確狀態機。

---

## 十三、最終建議

第一版應實作「單一保留區間、非破壞式、預設精準修剪」。

最適合的產品流程是先修剪再轉錄，原因是：

- 可減少 Whisper 處理時間。
- 可減少字幕段落與人工校對量。
- 字幕天然從修剪後影片 0 秒開始。
- 對現有架構的改動與錯誤風險最低。

既有字幕後修剪應同一版支援基本的時間重算，但多段刪除與影片拼接應延後到第二階段，避免第一版變成難以維護的完整剪輯器。
