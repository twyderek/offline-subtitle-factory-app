# 測試與驗收流程

## 輸入

- 需求驗收準則、變更差異、風險等級、測試環境。

## 步驟

1. 建立需求 ID 到測試案例／指令的對照。
2. 先執行最接近變更的測試，再執行 `npm run check`。
3. 測試成功、失敗、取消、逾時、空值、邊界、重啟與不支援輸入。
4. UI 變更以實際畫面驗證；媒體輸出以 ffprobe／播放器驗證。
5. 封裝變更檢查 unpacked 與最終安裝包，不只檢查 source。
6. 記錄環境、指令、預期、實際與未覆蓋項目至 `06-TEST-AND-PROCESS-AUDIT.md` 或工作紀錄。

## 輸出

- 可重現測試證據、通過／失敗判定、未覆蓋風險。

## 停止條件

- 核心回歸失敗、checksum 不一致、資料遺失、secrets 洩漏、假成功狀態或無法解釋的環境差異。
