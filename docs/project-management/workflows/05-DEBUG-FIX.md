# 偵錯與修正流程

## 輸入

- 現象、錯誤訊息、版本、環境、最小重現步驟與相關檔案。

## 步驟

1. 在工作紀錄建立項目，於 `07-DEBUG-AND-FIX-HISTORY.md` 配置 `BUG-ID`。
2. 先重現並保存基準證據，不以猜測直接修改。
3. 區分症狀、近因、根因與環境因素。
4. 設計最小修正、回復方式與防回歸測試。
5. 驗證原重現已消失、鄰近功能無回歸、封裝環境與 source 結果一致。
6. 記錄根因、修正、驗證與仍存在的 workaround。

## 輸出

- 可重現根因、最小修正、防回歸測試與完整 BUG 紀錄。

## 停止條件

- 無法重現且缺乏足夠診斷資料、修正需破壞資料、或需要擴大權限／外部變更。
