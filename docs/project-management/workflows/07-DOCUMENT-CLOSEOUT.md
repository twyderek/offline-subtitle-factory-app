# 文件同步與結案流程

## 輸入

- 實際差異、測試、審查、部署結果、已知風險與後續事項。

## 步驟

1. 依責任矩陣只更新實際受影響文件；不要求每次重讀或改寫無關歷史文件。
2. 補齊 `08-CHANGE-LOG.md`，不得留下「待執行」卻宣稱完成；「待確認」僅可用於尚待人類查證的事實或歷史缺口，且須在遺留風險寫明影響與追蹤方式。
3. 確認計畫功能與現有功能用語分離。
4. 執行 `npm run docs:check:final`、`git diff --check` 與必要回歸。
5. 確認審查代理自行建立的獨立報告存在，工作紀錄只連結並逐字引用判定；發布等級另確認具體發布授權已記錄。
6. 最終交付分開呈現開發驗證與獨立審查。

## 輸出

- 與實際狀態一致、可由下一位執行者直接使用的治理文件與工作紀錄。

## 停止條件

- 文件與程式／Release 衝突、缺少證據、工作紀錄未結案或仍有未揭露風險。
