# 需求新增與變更流程

## 輸入

- 使用者需求、問題證據、現有需求 ID、版本與限制。

## 步驟

1. 在 `08-CHANGE-LOG.md` 建立工作條目。
2. 將需求拆成可驗收結果，確認使用者、情境、成功條件與不在範圍。
3. 在 `02-REQUIREMENTS-ANALYSIS.md` 新增或修訂 `FR`／`NFR` ID。
4. 分析隱私、安全、資料、相容性、部署與回復影響。
5. 若有多種合理方向且會 materially 改變成果，取得需求方決定。
6. 將需求 ID 連結至功能設計、測試與工作紀錄。

## 輸出

- 可追溯需求 ID、明確驗收準則、範圍與風險。

## 停止條件

- 核心需求互相衝突、必要授權缺失、或選項會造成實質不同結果且無法由現況推定。
