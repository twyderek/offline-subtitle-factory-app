# 軟體開發流程

## 輸入

- 已核准需求 ID、設計影響、工作計畫與驗證計畫。

## 步驟

1. 執行 preflight，檢查 Git 狀態並辨識使用者既有變更。
2. 更新 `03-FUNCTIONAL-DESIGN.md` 中受影響的模組、資料流與錯誤處理。
3. 小步實作，每個修改維持需求範圍，不重寫無關程式。
4. 新行為同步新增或更新自動測試；無法自動化時預先定義人工證據。
5. 執行相關快速測試，修正後再執行完整 `npm run check`。
6. 更新工作紀錄的實際修改、決策與偏離原計畫原因。

## 輸出

- 可測試的程式碼、同步設計與測試、清楚差異。

## 停止條件

- 發現未授權的範圍擴張、資料破壞風險、secrets、無法安全遷移或與既有使用者修改衝突。
