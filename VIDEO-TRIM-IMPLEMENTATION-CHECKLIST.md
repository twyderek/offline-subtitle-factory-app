# 簡易影片修剪：實作工作步驟

> 整理日期：2026-07-16
> 對應規格：`VIDEO-TRIM-FEATURE-PLAN.md`
> 第一版範圍：單一保留區間、非破壞式、精準修剪、轉錄前修剪與既有字幕時間重算。

---

## 執行原則

- 每完成一項才進入下一項，並將結果寫入 `NEXT-VERSION-FIX-LOG.md`。
- 每個階段先完成自動測試，再進入 UI 或封裝測試。
- 不覆蓋 `input` 原始影片。
- 所有修剪輸出先寫入 `.partial`，驗證成功後才更名。
- 第一版不加入多段剪接、轉場、濾鏡、多軌與素材拼接。

## 階段 0：建立安全基準

### 步驟 0.1：確認目前版本基準

工作內容：

- 執行現有 `npm run check`。
- 確認首頁、字幕轉錄、校閱、真實聲波與輸出測試通過。
- 記錄目前版本、測試影片與預期輸出。

驗收條件：

- 尚未加入修剪前，所有既有功能測試均通過。
- 發生後續退步時，可以和此基準比較。

日誌節點：`修剪功能－基準測試完成`。

## 階段 1：資料模型與純邏輯

### 步驟 1.1：建立 edit plan 規格

預計修改／新增：

- `server.mjs`
- 建議新增 `lib/media-edit.mjs`
- `scripts/test-core.mjs`

工作內容：

- 定義 `edit-plan.json` version 1。
- 欄位包含 `mode`、`sourceDuration`、`in`、`out`、`outputDuration`、`strategy`、`createdAt`。
- 建立伺服器端驗證：
  - `in >= 0`
  - `out <= sourceDuration`
  - `out > in`
  - 最短保留 0.5 秒
  - strategy 只允許 `precise` 或 `fast`

驗收條件：

- 合法資料可正規化並保存。
- 負數、超過影片長度、起點大於終點與過短片段均被拒絕。

### 步驟 1.2：抽象化影片來源

工作內容：

- 將目前 `getVideoPath(job)` 拆為：
  - `getOriginalVideoPath(job)`
  - `getEffectiveVideoPath(job)`
- 尚未完成修剪時，effective path 必須回退原始影片。
- `.partial` 不可被當成有效影片。

驗收條件：

- 未修剪專案完全維持原行為。
- 修剪完成後能正確選到 `working/media-trimmed.mp4`。
- 還原修剪後重新使用原始影片。

### 步驟 1.3：建立字幕時間重算函式

建議新增：

- `lib/subtitle-timeline.mjs`

工作內容：

- 實作單一保留區間字幕轉換。
- 移除區間外字幕。
- 截短跨越 In／Out 的字幕。
- 重新編號與重建 SRT。
- 小於 0.1 秒的字幕刪除。
- 邊界字幕回傳 `needsReview` 標記。

驗收條件：

- 純函式單元測試涵蓋完整保留、完全刪除、跨起點、跨終點及空字幕。

日誌節點：`修剪功能－資料層與字幕時間軸完成`。

## 階段 2：後端 API 與狀態機

### 步驟 2.1：建立 edit plan API

新增端點：

```text
GET /api/jobs/:jobId/edit-plan
PUT /api/jobs/:jobId/edit-plan
DELETE /api/jobs/:jobId/trim
```

工作內容：

- GET 讀取目前設定與原始影片時長。
- PUT 驗證並保存 edit plan。
- DELETE 停用修剪結果、恢復原始影片並清除衍生快取。

驗收條件：

- API token 與 Origin 防護沿用現有規則。
- 不可透過 jobId 或路徑欄位存取任務外檔案。

### 步驟 2.2：建立 FFmpeg 精準修剪

新增端點：

```text
POST /api/jobs/:jobId/apply-trim
GET  /api/jobs/:jobId/trim-status
POST /api/jobs/:jobId/cancel-trim
```

工作內容：

- 新增 `runningTrims` 與 `AbortController`。
- 先完成 precise 模式。
- macOS 優先測試 `h264_videotoolbox`，失敗回退 `libx264`。
- 音訊輸出 AAC，保留原解析度與比例。
- 使用 FFmpeg `-progress pipe:1` 回報真實進度。
- 完成後用 FFprobe 驗證：
  - 檔案可讀
  - 有視訊軌
  - 長度符合選取區間
  - 時間戳從接近 0 開始
- 驗證成功後將 `.partial.mp4` 原子更名。

驗收條件：

- 10 秒影片修剪 2–7 秒後，輸出約為 5 秒。
- 影片與音訊可正常播放。
- 中止後 FFmpeg 結束且 partial 被清除。

### 步驟 2.3：加入快速修剪

工作內容：

- 加入 stream copy 模式。
- UI/API 回傳 keyframe 精準度提示。
- 若來源格式無法直接封裝，顯示可理解錯誤並建議使用精準模式。

驗收條件：

- 快速模式不重新編碼可相容來源。
- 不宣稱其邊界為逐幀精準。

### 步驟 2.4：任務鎖與錯誤處理

工作內容：

- 同一任務不可同時修剪、轉錄或燒錄。
- 增加磁碟空間預估與可寫入檢查。
- App 關閉或重啟後清理過期 partial。
- 新增 `trimming`、`trim-completed`、`trim-failed` 等狀態。

驗收條件：

- 衝突操作回傳 409，不造成兩個 FFmpeg 同時改寫任務。
- 空間不足時在處理前停止。

日誌節點：`修剪功能－後端與 FFmpeg 完成`。

## 階段 3：修剪工作區 UI

### 步驟 3.1：建立獨立修剪頁

建議新增：

- `public/trim.html`
- `public/trim.js`
- `public/styles.css` 修剪區樣式
- server route：`/trim/:jobId`

採獨立頁面的原因：

- 避免繼續擴大目前的 `review.js`。
- 修剪與字幕校閱可分別測試。
- 首頁、新增專案與校閱頁都能導向同一工作區。

工作內容：

- 沿用深藍側欄與現有品牌樣式。
- 顯示影片、波形、時間軸、設定面板與進度。
- 提供返回專案與前往字幕校對。

驗收條件：

- 可以載入 job 的 effective media 與 edit plan。
- 版面在 1280×720、1440×900 與目前 Electron 視窗尺寸正常。

### 步驟 3.2：時間軸與修剪把手

工作內容：

- 重用真實聲波 API。
- 加入 In／Out 左右把手與刪除區遮罩。
- 拖曳時同步預覽畫面與數字時間。
- 支援時間欄位直接輸入。
- 防止兩個把手交錯。

驗收條件：

- 把手、數字欄位、播放游標三者狀態一致。
- 長影片拖曳仍流暢，不在 pointermove 呼叫後端。

### 步驟 3.3：播放與快捷鍵

工作內容：

- I 設定起點、O 設定終點。
- Space 播放／暫停。
- 前後 5 秒。
- 循環播放選取區間。
- 顯示原長、保留長度、刪除長度。

驗收條件：

- 編輯輸入框時快捷鍵不會誤觸播放器。
- 循環播放不超出 Out。

### 步驟 3.4：套用、取消與還原

工作內容：

- 套用前顯示非破壞式說明與模式。
- 顯示 FFmpeg 真實進度與取消按鈕。
- 成功後重新載入影片、聲波、縮圖與時間。
- 提供還原原始影片。

驗收條件：

- 不重整頁面也能看到修剪後結果。
- 取消或失敗不會把 partial 當成完成檔案。

日誌節點：`修剪功能－UI 與預覽完成`。

## 階段 4：既有流程整合

### 步驟 4.1：新增專案轉錄前修剪

工作內容：

- 選擇影片後提供「先修剪影片」。
- 建立 job 時保存 In／Out。
- 若有 edit plan，先完成修剪，再執行音訊前處理與 Whisper。
- 修剪失敗時禁止直接進入轉錄完成狀態。

驗收條件：

- Whisper 實際讀取 trimmed media。
- 轉錄字幕從 0 秒開始。
- 原始影片仍存在 input。

### 步驟 4.2：既有字幕專案修剪

工作內容：

- 修剪前建立字幕備份。
- 套用單區間字幕時間重算。
- 邊界字幕標記待檢查。
- 輸出 `review-output/trimmed.srt`。

驗收條件：

- 校閱頁影片與字幕保持同步。
- 還原時可恢復原影片及原字幕版本。

### 步驟 4.3：所有下游功能改用 effective media

逐一檢查：

- 影片資訊／FFprobe。
- 真實聲波。
- 首頁縮圖。
- 校閱播放器。
- 音訊前處理與 Whisper。
- 軟字幕輸出。
- 硬字幕燒錄。

驗收條件：

- 修剪後所有畫面與輸出都使用同一支有效影片。
- 未修剪專案沒有行為改變。

### 步驟 4.4：快取失效

工作內容：

- 套用或還原修剪時刪除舊 waveform、thumbnail 與 probe cache。
- 新快取需能區分原影片與修剪影片版本。

驗收條件：

- 首頁不會顯示舊縮圖。
- 校閱頁不會顯示舊影片長度或舊聲波。

日誌節點：`修剪功能－字幕與完整流程整合完成`。

## 階段 5：測試與效能

### 步驟 5.1：自動測試

加入測試：

- edit plan validation。
- effective media path。
- 字幕時間重算。
- precise／fast FFmpeg 修剪。
- 取消與 partial 清除。
- 任務鎖。
- cache invalidation。
- API token／Origin。

驗收條件：

- `npm run check` 全部通過。

### 步驟 5.2：真實影片測試

測試組合：

- 短影片：10–30 秒。
- 長影片：目前 11:33 測試影片。
- 有聲音影片。
- 沒有音軌影片。
- 可變幀率影片。
- MOV、MP4、M4V。

記錄：

- 修剪耗時。
- CPU／GPU 模式。
- 輸出大小。
- 長度誤差。
- 音畫同步。

### 步驟 5.3：UI 自動驗收

檢查：

- 修剪頁可見。
- 把手與時間輸入同步。
- 點擊波形跳轉。
- I／O 快捷鍵。
- 進度、取消與還原。
- 修剪後前往校閱。

日誌節點：`修剪功能－自動與真實影片測試完成`。
